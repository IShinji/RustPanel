# RustPanel 任务清单 v3.0 - 与宝塔免费版功能对齐

更新日期：2026-05-09

## 版本目标

补齐 RustPanel 在**可视化安全、进阶站点、自动化运维、监控历史**方面与宝塔免费版的代差。实现真正“开箱即用”且具备“生产防御能力”的运维面板。

## 对标参考：宝塔面板核心功能概览

以下是作为对标参考的宝塔面板核心功能总结：

### 1. 状态实时监控
* **系统负载与资源占用**：展示服务器的负载状态、CPU 使用率、内存占用以及磁盘根目录占用。
* **流量与 IO 统计**：提供实时的网络流量图表（上行/下行速度）和磁盘 I/O 读写监控。

### 2. 网站与数据库管理
* **站点管理**：支持域名绑定、目录设置、伪静态配置等。
* **数据库管理**：支持 MySQL 等数据库的创建、备份和导入导出。

### 3. 系统安全与防御
* **风险预警**：提示系统安全风险，引导进行系统加固。
* **安全模块**：集成 WAF 防火墙、SSH 管理、端口放行等功能。

### 4. 软件与环境部署
* **应用商店**：一键安装 Nginx、Apache、PHP、Node.js 等开发环境及各类工具插件。
* **容器化支持**：包含 Docker 管理，方便运行容器应用。

### 5. 系统工具
* **文件管理**：提供 Web 端操作界面，支持在线编辑代码、上传下载文件。
* **计划任务**：设置定时备份、定期重启或执行自定义脚本。
* **终端**：内置 Web 端 SSH 终端。

**参考环境信息：**
* **操作系统**：Debian 13 企业版
* **面板版本**：v11.7.0

## P1 第一步：安全底座与防火墙 (Security & Firewall)

目标：可视化接管宿主机网络安全，保护面板不被暴力破解。

### P1-01 系统防火墙 UI 化
- [ ] **P1-01-1** 定义 `security.proto`：包含端口放行/封禁规则、协议选择（TCP/UDP）、备注说明。
- [ ] **P1-01-2** 后端实现适配层：兼容 `ufw` (Ubuntu/Debian) 和 `firewalld` (CentOS/RHEL)。
- [ ] **P1-01-3** 前端开发“安全管理”页面：展示当前规则列表，支持增删改规则。

### P1-02 SSH 安全增强
- [ ] **P1-02-1** 实现 SSH 端口修改 RPC 及后端逻辑（需同步修改 `/etc/ssh/sshd_config` 并重启服务）。
- [ ] **P1-02-2** 实现 SSH 登录审计：记录并展示最近 100 条登录成功/失败的记录。
- [ ] **P1-02-3** 实现 SSH 密钥管理：在线生成 RSA/Ed25510 密钥对，支持禁用密码登录。

---

## P2 第二步：进阶 Web 运维 (Advanced Web Server)

目标：精细化 Nginx 配置，满足复杂建站需求。

### P2-01 伪静态与重定向
- [ ] **P2-01-1** 扩展 `site.proto`，支持 URL 重写 (Rewrite) 和 HTTP 重定向 (Redirect) 的配置协议。
- [ ] **P2-01-2** 内置常用伪静态模板库：WordPress, Laravel, ThinkPHP, Discuz! 等。
- [ ] **P2-01-3** 前端实现“伪静态”编辑器：支持模板选择和手动输入 Nginx 指令。

### P2-02 进阶代理与 SSL 闭环
- [ ] **P2-02-1** 实现站点级 IP 黑白名单配置。
- [ ] **P2-02-2** 实现子目录反向代理配置界面。
- [ ] **P2-02-3** 完善 SSL 自动续期：后台定时检测证书有效期（< 30天），自动触发 `acme-lib` 续签流程并重载 Nginx。

## P3 第三步：文件管理与运维工具 (Files & Utilities)

目标：提升日常文件处理效率，降低误操作风险。

### P3-01 远程下载与回收站
- [ ] **P3-01-1** 实现“远程下载”功能：输入 URL 直接将文件下载到指定目录，后端显示下载进度。
- [ ] **P3-01-2** 实现“回收站”功能：删除文件时移动至 `.rustpanel_trash` 目录并保留原路径信息。
- [ ] **P3-01-3** 前端开发“回收站”管理模块：支持一键清空或按需还原。

### P3-02 全局搜索与替换
- [ ] **P3-02-1** 利用 `ripgrep` (如果环境有) 或后端原生实现目录级的内容搜索。
- [ ] **P3-02-2** 前端编辑器支持全局内容搜索与批量替换。

## P4 第四步：监控历史与性能看板 (Monitoring History)

目标：不仅看到当下，更能溯源异常。

### P4-01 时序数据持久化
- [ ] **P4-01-1** 引入 SQLite 或嵌入式时序存储，记录每分钟的 CPU、内存、IO、网络指标。
- [ ] **P4-01-2** 实现数据自动滚动清理机制：默认保留最近 30 天的历史数据。
- [ ] **P4-01-3** 前端 Dashboard 增加“历史趋势”图表：支持查看 1h, 24h, 7d 的统计曲线。

## P5 第五步：自动化维护任务 (Auto-Maintenance)

目标：将重复劳动交给系统。

### P5-01 任务模板扩展
- [ ] **P5-01-1** 增加“日志切割/清理”任务模板：自动清理 Nginx 日志、系统临时文件。
- [ ] **P5-01-2** 增加“URL 监控任务”：定时访问特定 URL，记录响应状态码。
- [ ] **P5-01-3** 增加“站点/数据库自动备份”至异地（S3/R2/OSS）的完整流程闭环。
