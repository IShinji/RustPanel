# RustPanel 任务清单 v2.0 - 现代架构演进与功能闭环

更新日期：2026-05-09

## 版本目标

抛开旧有架构包袱，完全以**架构演进和功能闭环**的逻辑重构/开发 RustPanel。对标宝塔免费功能，但底层全面现代化。采用全栈架构：Rust (Tonic + Axum) + gRPC (Protobuf) + React + Docker 容器化。

## 执行规则

- 按 `blocker → high → medium → low` 执行。
- 完成后将 `[ ]` 改为 `[x]`。
- 阻塞任务保持 `[ ]` 并添加 `@status: BLOCKED`。
- 本地持续利用 AI Agentic Tools 和 M4 Max 本地算力飞速编译和验证容器行为，极度压缩样板代码时间。

## P1 基础交付 - 第一步：通信底座与系统守护 (Foundation & Daemon)

目标：建立整个面板的骨架，实现极简的单文件部署。

### P1-01 核心通信协议 (Protobuf)
- [ ] **P1-01-1** 初始化 `proto` 目录与基础 Protobuf 环境
  - 优先级：blocker
  - 验收标准：
    - 安装并配置 `protoc` 编译器依赖。
    - 引入 `tonic-build` 到 Rust 后端 `build.rs`。
- [ ] **P1-01-2** 定义基础请求响应封装 (Base Request/Response)
  - 优先级：high
  - 验收标准：
    - 统一定义包含 code, message, data 字段的通用响应体。
- [ ] **P1-01-3** 编写系统健康与心跳探测服务 Proto (`health.proto`)
  - 优先级：high
  - 验收标准：
    - 实现 `HealthCheck` RPC 接口，验证 gRPC 信道畅通。

### P1-02 HTTP/gRPC 混合服务端 (Axum + Tonic / Salvo)
- [ ] **P1-02-1** 搭建联合路由架构
  - 优先级：blocker
  - 验收标准：
    - 同一个端口能同时接收 HTTP (REST/静态资源) 请求和 gRPC 请求。
- [ ] **P1-02-2** 集成 JWT 鉴权中间件 (gRPC Interceptor / HTTP Middleware)
  - 优先级：high
  - 验收标准：
    - 调用能拦截并校验 Token，未授权请求返回 `UNAUTHENTICATED` 状态码。
- [ ] **P1-02-3** 面板守护进程初始化逻辑 (Daemon)
  - 优先级：high
  - 验收标准：
    - 进程支持后台运行。
    - 配合系统级服务配置，崩溃可自动拉起重启。

### P1-03 静态资源内嵌打包 (rust-embed)
- [ ] **P1-03-1** 前端 React 骨架初始化及工程化配置
  - 优先级：high
  - 验收标准：
    - 完成前端 React + Vite 基础环境搭建。
    - 配置 gRPC-web 客户端代码自动生成通道。
- [ ] **P1-03-2** 配置 `rust-embed` 内联前端静态资源
  - 优先级：high
  - 验收标准：
    - 将编译后的前端 `dist` 直接作为二进制的一部分打包。
- [ ] **P1-03-3** 极简部署验证
  - 优先级：high
  - 验收标准：
    - 停止所有服务，仅运行编译后的单一 Rust 二进制应用，浏览器可正常访问并交互。

## P2 第二步：资源监控与 Web 终端 (Monitoring & Terminal)

目标：实时可视化展示服务器状态，并在浏览器中接管底层终端。

### P2-01 系统状态采集 (`sysinfo`)
- [ ] **P2-01-1** 封装 `sysinfo` 数据采集服务
  - 优先级：high
  - 验收标准：
    - 高效获取 CPU、内存、系统负载、磁盘 IO 和网卡流量数据。
- [ ] **P2-01-2** 实时数据推流服务 (gRPC Server Streaming / WebSocket)
  - 优先级：high
  - 验收标准：
    - 建立从后端向前端高频 (如每秒更新) 推送系统级资源占用情况的流通道。
- [ ] **P2-01-3** 前端监控仪表盘面板 (Dashboard)
  - 优先级：high
  - 验收标准：
    - 前端订阅流数据，通过动态图表平滑展示资源变化。

### P2-02 极速 Web SSH (`portable-pty` + `xterm.js`)
- [ ] **P2-02-1** PTY 派生与进程管理
  - 优先级：blocker
  - 验收标准：
    - 基于 `portable-pty` 安全启动 Bash/Zsh 伪终端进程。
    - 将 PTY 的字节流接入 gRPC 流或 WebSocket。
- [ ] **P2-02-2** 前端终端模拟器 (`xterm.js`)
  - 优先级：high
  - 验收标准：
    - 浏览器中集成 `xterm.js`，保证色彩渲染与原生一致。
    - 同步窗口大小 (Resize) 使行列格式不崩坏。
- [ ] **P2-02-3** 会话与审计
  - 优先级：medium
  - 验收标准：
    - 记录管理员登录开启终端的时间，保证多会话隔离。

## P3 第三步：可视化文件系统 (File Manager)

目标：对标核心常用的文件管理功能，确保处理深层级和海量文件的高性能。

### P3-01 异步文件树操作
- [ ] **P3-01-1** 基于 `tokio::fs` 的基础 IO
  - 优先级：high
  - 验收标准：

- [ ] **P3-01-2** 极速深层遍历 (`walkdir` / `ignore`)
  - 优先级：high
  - 验收标准：
    - 秒级响应深层级包含大量小文件的目录（如 `node_modules`）结构树。
- [ ] **P3-01-3** 权限与属性修改
  - 优先级：high
  - 验收标准：
    - 支持 UI 上通过 chmod/chown 变更所有权与读写执行权限。

### P3-02 高阶文件处理与在线编辑
- [ ] **P3-02-1** 大文件流式上传下载
  - 优先级：high
  - 验收标准：
    - GB级大文件传输分块写入，保持极低内存占用。
- [ ] **P3-02-2** 打包与解压 (`zip`, `tar`, `flate2`)
  - 优先级：high
  - 验收标准：
    - 面板支持后台压缩目录为 `tar.gz`/`zip`，解压同理，提供进度状态。
- [ ] **P3-02-3** 在线代码编辑器 (`Monaco Editor`)
  - 优先级：medium
  - 验收标准：
    - 集成 Monaco 引擎，支持主流代码文件的顺滑编辑、语法高亮和保存。

## P5 第五步：站点与反向代理中心 (Web Server & Proxy)

目标：极简的 Nginx 配置和 SSL 自动化体验。

### P5-01 站点配置管理引擎
- [ ] **P5-01-1** Nginx 配置渲染模板 (`Tera` / `Askama`)
  - 优先级：high
  - 验收标准：
    - 通过接口参数瞬间渲染生成带反代、PHP、静态资源设定的 Nginx `.conf` 结构。
- [ ] **P5-01-2** 配置平滑重载
  - 优先级：high
  - 验收标准：
    - 自动触发 `nginx -t` 检测，通过后无缝 `nginx -s reload`。
- [ ] **P5-01-3** 可视化高级规则
  - 优先级：medium
  - 验收标准：
    - 直观的添加页面跳转(301/302)，SSL强制跳转和防盗链配置。

### P5-02 SSL 自动化
- [ ] **P5-02-1** 自动申请 Let's Encrypt 证书 (`acme-lib`)
  - 优先级：high
  - 验收标准：
    - 完成 HTTP-01 或 DNS-01 验证并下发保存私钥及证书。
- [ ] **P5-02-2** 后台自动续期
  - 优先级：high
  - 验收标准：
    - 基于调度器进行证书临近到期巡检，到期前自动更新并重载 web server。

  - 验收标准：
